## Callout

Use callouts to highlight important information. Supports four types: `info`, `warning`, `error`, and default.

```mdx
import { Callout } from 'nextra/components'

<Callout type="info">This is an info callout.</Callout>
<Callout type="warning">This is a warning.</Callout>
<Callout type="error">This is an error or danger notice.</Callout>
<Callout>Default callout (no type).</Callout>
```

**Result:**

<Callout type="info">This is an **info** callout — great for tips or background context.</Callout>
<Callout type="warning">This is a **warning** — use for potential issues or caveats.</Callout>
<Callout type="error">This is an **error** — use for dangerous actions or critical notes.</Callout>
<Callout>This is a **default** callout — neutral emphasis.</Callout>

## Steps

Use `Steps` for sequential instructions. Each `###` heading inside becomes a numbered step.

```mdx
import { Steps } from 'nextra/components'

<Steps>
### Step One
Do the first thing.

### Step Two
Do the second thing.

### Step Three
Done!
</Steps>
```

**Result:**

<Steps>
### Step One
Do the first thing. You can include any markdown content here — paragraphs, code blocks, callouts.

### Step Two
Do the second thing.

### Step Three
Done!
</Steps>

## Tabs

Use `Tabs` to present alternative content or compare options (e.g. package manager choices).

```mdx
import { Tabs } from 'nextra/components'

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tabs.Tab>pnpm install</Tabs.Tab>
  <Tabs.Tab>npm install</Tabs.Tab>
  <Tabs.Tab>yarn install</Tabs.Tab>
</Tabs>
```

**Result:**

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tabs.Tab>
    ```bash
    pnpm install
    pnpm dev
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    npm install
    npm run dev
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    yarn install
    yarn dev
    ```
  </Tabs.Tab>
</Tabs>

## Cards

Use `Cards` for navigation or highlighting related links in a card layout.


## FileTree

Use `FileTree` to visually represent a folder/file structure. Folders are interactive — click to expand/collapse.

```mdx
import { FileTree } from 'nextra/components'

<FileTree>
  <FileTree.Folder name="content" defaultOpen>
    <FileTree.File name="_meta.js" />
    <FileTree.File name="index.mdx" />
    <FileTree.Folder name="guides">
      <FileTree.File name="_meta.js" />
      <FileTree.File name="index.mdx" />
    </FileTree.Folder>
  </FileTree.Folder>
</FileTree>
```

**Result:**

<FileTree>
  <FileTree.Folder name="nextra4-starter" defaultOpen>
    <FileTree.Folder name="app" defaultOpen>
      <FileTree.File name="layout.tsx" />

## Table

```mdx

<Table>
  <thead>
    <Table.Tr>
      <Table.Th>Country</Table.Th>
      <Table.Th>Flag</Table.Th>
    </Table.Tr>
  </thead>
  <tbody>
    <Table.Tr>
      <Table.Td>France</Table.Td>
      <Table.Td>🇫🇷</Table.Td>
    </Table.Tr>
  </tbody>
</Table>
```

## Bleed

Use `Bleed` to make content slightly exceed the container width — useful for images, banners, or wide elements.

```mdx
import { Bleed } from 'nextra/components'

{/* Normal bleed — exceeds container padding */}
<Bleed>
  <div>Content wider than the text column</div>
</Bleed>

{/* Full bleed — extends to the full screen edge */}
<Bleed full>
  <div>Full-width content</div>
</Bleed>
```

**Result:**

<Bleed>
  <div style={{ background: 'var(--x-color-primary-200, #e0f2fe)', padding: '1.5rem', textAlign: 'center', borderRadius: '0.5rem', fontWeight: 600 }}>
    🌊 Normal Bleed — content exceeds container padding
  </div>
</Bleed>

<Bleed full>
  <div style={{ background: 'var(--x-color-primary-800, #1e3a5f)', color: 'white', padding: '1.5rem', textAlign: 'center', fontWeight: 600, marginTop: '1rem' }}>
    🌊 Full Bleed — content extends to full screen width
  </div>
</Bleed>

## Code Blocks

Fenced code blocks with syntax highlighting are built-in. Add `filename` to show a file label, and highlight specific lines with `{n}`.

````mdx
```tsx filename="app/layout.tsx" {3}
export default function Layout({ children }) {
  return (
    <div>{children}</div>   // ← this line is highlighted
  )
}
```
````

**Result:**

```tsx filename="app/layout.tsx" {3}
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>{children}</div>
  )
}
```

## Banner (Layout Component)

`Banner` appears at the very top of the site, above the navbar. Enable it in `app/[lang]/layout.tsx`:

```tsx filename="app/[lang]/layout.tsx"
import { Banner, Head } from 'nextra/components'

// Inside the return:
<Banner storageKey="release-v1" dismissible>
  🎉 Template v1.0 released! <a href="/guides">View the guide →</a>
</Banner>
```

Available props:

| Prop | Type | Default | Description |
| :--- | :--- | :--- | :--- |
| `dismissible` | `boolean` | `true` | Whether the banner can be closed by the user |
| `storageKey` | `string` | `'nextra-banner'` | localStorage key to persist the dismissed state |

## Combining Components

All components can be freely nested:

<Tabs items={['Example', 'Code']}>
  <Tabs.Tab>
    <Callout type="info">
      You can put a **Callout** inside a **Tab**, or a **FileTree** inside **Steps**. Mix and match freely!
    </Callout>
    <Cards>
      <Cards.Card title="Nested Card" href="#" description="Cards inside tabs work great for contextual navigation." />
    </Cards>
  </Tabs.Tab>
  <Tabs.Tab>
    ```mdx
    <Tabs items={['Example', 'Code']}>
      <Tabs.Tab>
        <Callout type="info">Callout inside a Tab!</Callout>
        <Cards>
          <Cards.Card title="Nested Card" href="#" description="Works!" />
        </Cards>
      </Tabs.Tab>
    </Tabs>
    ```
  </Tabs.Tab>
</Tabs>
