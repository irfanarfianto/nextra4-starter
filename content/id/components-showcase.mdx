## Callout

Gunakan callout untuk menyoroti informasi penting. Mendukung empat tipe: `info`, `warning`, `error`, dan default.

```mdx
import { Callout } from 'nextra/components'

<Callout type="info">Ini adalah callout info.</Callout>
<Callout type="warning">Ini adalah peringatan.</Callout>
<Callout type="error">Ini adalah error atau bahaya.</Callout>
<Callout>Callout default (tanpa tipe).</Callout>
```

**Hasil:**

<Callout type="info">Ini adalah callout **info** — bagus untuk tips atau konteks latar belakang.</Callout>
<Callout type="warning">Ini adalah **peringatan** — gunakan untuk masalah potensial atau hal yang perlu diperhatikan.</Callout>
<Callout type="error">Ini adalah **error** — gunakan untuk tindakan berbahaya atau catatan kritis.</Callout>
<Callout>Ini adalah callout **default** — penekanan netral.</Callout>

## Steps

Gunakan `Steps` untuk instruksi berurutan. Setiap heading `###` di dalamnya menjadi satu langkah bernomor.

```mdx
import { Steps } from 'nextra/components'

<Steps>
### Langkah Pertama
Lakukan hal pertama.

### Langkah Kedua
Lakukan hal kedua.

### Langkah Ketiga
Selesai!
</Steps>
```

**Hasil:**

<Steps>
### Langkah Pertama
Lakukan hal pertama. Anda bisa menyertakan konten markdown apa pun — paragraf, kode, callout.

### Langkah Kedua
Lakukan hal kedua.

### Langkah Ketiga
Selesai!
</Steps>

## Tabs

Gunakan `Tabs` untuk menyajikan konten alternatif atau membandingkan opsi (misalnya pilihan package manager).

```mdx
import { Tabs } from 'nextra/components'

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tabs.Tab>pnpm install</Tabs.Tab>
  <Tabs.Tab>npm install</Tabs.Tab>
  <Tabs.Tab>yarn install</Tabs.Tab>
</Tabs>
```

**Hasil:**

<Tabs items={['pnpm', 'npm', 'yarn']}>
  <Tabs.Tab>
    ```bash
    pnpm install
    pnpm dev
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    npm install
    npm run dev
    ```
  </Tabs.Tab>
  <Tabs.Tab>
    ```bash
    yarn install
    yarn dev
    ```
  </Tabs.Tab>
</Tabs>

## Cards

```mdx
import { Cards } from 'nextra/components'
```

## FileTree

Gunakan `FileTree` untuk menampilkan struktur folder/file secara visual. Folder bisa diklik untuk buka/tutup.

```mdx
import { FileTree } from 'nextra/components'

<FileTree>
  <FileTree.Folder name="content" defaultOpen>
    <FileTree.File name="_meta.js" />
    <FileTree.File name="index.mdx" />
    <FileTree.Folder name="guides">
      <FileTree.File name="_meta.js" />
      <FileTree.File name="index.mdx" />
    </FileTree.Folder>
  </FileTree.Folder>
</FileTree>
```

**Hasil:**

<FileTree>
  <FileTree.Folder name="nextra4-starter" defaultOpen>
    <FileTree.Folder name="app" defaultOpen>
      <FileTree.File name="layout.tsx" />
      <FileTree.Folder name="[lang]" defaultOpen>
        <FileTree.File name="layout.tsx" />
        <FileTree.Folder name="[[...mdxPath]]">
          <FileTree.File name="page.tsx" />
        </FileTree.Folder>
      </FileTree.Folder>
    </FileTree.Folder>
    <FileTree.Folder name="content">
      <FileTree.Folder name="id">
        <FileTree.File name="_meta.js" />
        <FileTree.File name="index.mdx" />
        <FileTree.Folder name="guides">
          <FileTree.File name="_meta.js" />
          <FileTree.File name="index.mdx" />
        </FileTree.Folder>
      </FileTree.Folder>
      <FileTree.Folder name="en">
        <FileTree.File name="_meta.js" />
        <FileTree.File name="index.mdx" />
      </FileTree.Folder>
    </FileTree.Folder>

## Bleed

Gunakan `Bleed` untuk membuat konten sedikit melampaui lebar kontainer — berguna untuk gambar, banner, atau elemen lebar penuh.

```mdx
import { Bleed } from 'nextra/components'

{/* Bleed biasa — melampaui padding kontainer */}
<Bleed>
  <div style={{ background: '#e0f2fe', padding: '1rem', textAlign: 'center' }}>
    Ini konten Bleed biasa
  </div>
</Bleed>

{/* Full bleed — melampaui hingga tepi layar */}
<Bleed full>
  <div style={{ background: '#dcfce7', padding: '1rem', textAlign: 'center' }}>
    Ini konten full Bleed
  </div>
</Bleed>
```

**Hasil:**

<Bleed>
  <div style={{ background: 'var(--x-color-primary-200, #e0f2fe)', padding: '1.5rem', textAlign: 'center', borderRadius: '0.5rem', fontWeight: 600 }}>
    🌊 Bleed biasa — konten melampaui padding kontainer
  </div>
</Bleed>

<Bleed full>
  <div style={{ background: 'var(--x-color-primary-800, #1e3a5f)', color: 'white', padding: '1.5rem', textAlign: 'center', fontWeight: 600, marginTop: '1rem' }}>
    🌊 Full Bleed — konten melampaui hingga tepi layar
  </div>
</Bleed>

## Blok Kode

Blok kode dengan syntax highlighting sudah bawaan. Tambahkan `filename` untuk menampilkan nama file, dan gunakan highlight baris dengan `{n}`.

````mdx
```tsx filename="app/layout.tsx" {3}
export default function Layout({ children }) {
  return (
    <div>{children}</div>   // ← baris ini di-highlight
  )
}
```
````

**Hasil:**

```tsx filename="app/layout.tsx" {3}
export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <div>{children}</div>
  )
}
```

## Banner (Layout Component)

`Banner` ditampilkan di bagian paling atas situs, di atas navbar. Aktifkan di `app/[lang]/layout.tsx`:

```tsx filename="app/[lang]/layout.tsx"
import { Banner, Head } from 'nextra/components'

// Di dalam return:
<Banner storageKey="release-v1" dismissible>
  🎉 Template v1.0 sudah dirilis! <a href="/guides">Lihat panduan →</a>
</Banner>
```

Props yang tersedia:

| Prop | Tipe | Default | Deskripsi |
| :--- | :--- | :--- | :--- |
| `dismissible` | `boolean` | `true` | Banner bisa ditutup pengguna |
| `storageKey` | `string` | `'nextra-banner'` | Key localStorage untuk menyimpan state tutup |

    <Tabs items={['Contoh', 'Kode']}>

        <Cards>

      </Tabs.Tab>

  </Tabs.Tab>
